# OctoAcme Project Management Docs

> This README provides a centralized guide to OctoAcme's project management documentation, making it easier to discover, navigate, and onboard to our processes. Closes #2.

---

## Overview

OctoAcme's project management approach is built on the following principles:

- **Customer-focused delivery** — prioritize customer value and usability in every decision.
- **Iterative improvements** — deliver work in small, testable increments and learn continuously.
- **Ownership clarity** — every project has a named Project Manager (PM) and Product Lead accountable for outcomes.
- **Risk management** — risks and dependencies are identified early, tracked in a Risk Register, and reviewed regularly.
- **Stakeholder collaboration** — structured communication cadences keep all stakeholders informed and aligned.

### Lifecycle Phases

| Phase | Description |
|---|---|
| **Initiation** | Validate the problem, align stakeholders, and decide go/no-go |
| **Planning** | Break work into shippable increments, estimate, and create a release plan |
| **Execution** | Build, test, review, and track progress daily |
| **Release** | Deploy safely, verify, and announce to stakeholders |
| **Retrospective** | Capture learnings and convert them into continuous improvements |

## Onboarding

If you are new to OctoAcme, start here:

1. Read the **[Project Management Overview](./octoacme-project-management-overview.md)** to understand the team's principles, key roles, and common artifacts (Project Charter, Risk Register, Retrospective notes, etc.).
2. Review **[Roles & Personas](./octoacme-roles-and-personas.md)** to understand your responsibilities and how you interact with other roles.
3. When kicking off a new project, follow the **[Project Initiation Guide](./octoacme-project-initiation.md)** to produce a Project One-pager, stakeholder list, and initial risk list before moving to planning.
4. Use the remaining documents as references during the appropriate lifecycle phase.

---

## Key Artifacts

The following artifacts are referenced across the process documents:

- **Project Charter / One-pager** — created during Initiation.
- **Roadmap and Release Plan** — produced during Planning.
- **Sprint/Iteration Backlog** — maintained throughout Execution.
- **Risk Register** — initiated during Planning and updated weekly.
- **Definition of Done (DoD)** — agreed during Planning, enforced during Execution.
- **Release Notes** — drafted before every release.
- **Retrospective Notes & Action Items** — captured after each sprint, release, or milestone.
